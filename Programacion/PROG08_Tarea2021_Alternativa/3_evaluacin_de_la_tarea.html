3.- Evaluación de la tarea

Criterios de evaluación implicados

Peggy_Marco (Pixabay License)
  • Se han aplicado formatos en la visualización de la información.
  • Se han utilizado estructuras de datos avanzadas para almacenar la información gestionada por la aplicación: listas, conjuntos, árboles y mapas.
  • Se han utilizado clases genéricas.
  • Se han aplicado en el contexto adecuado las estructuras avanzadas de datos vistas en la unidad.
  • Se han utilizado clases y métodos propios del API de Java.
  • Se ha utilizado y explorado la documentación del API de Java en el contexto de un problema concreto.

¿Cómo valoramos y puntuamos tu tarea?

Rúbrica de la tarea

Ejercicio 1: Combinación aleatoria.

  • Se utiliza una estructura adecuada, de entre las estructuras avanzadas vistas en la unidad, para almacenar los valores.
  • Se recorre la estructura apropiadamente.
  • Se rellena la estructura con la información que se solicita en el enunciado.
  • Se muestran los números siguiendo el formato el indicado en el ejemplo (números separados por espacios, sin comas).

1 punto

Ejercicio 2: Vaciando la lista.

  • Se utilizan listas para almacenar los números.
  • La lista se rellena solo números entre 1 y 5, ambos incluidos.
  • La lista contiene exactamente la cantidad de números indicados por el usuario.
  • Se va obteniendo secuencialmente el valor del primer elemento de la lista.
  • Se eliminan de manera apropiada todas las apariciones del valor que aparece primero en la lista.
  • Se muestra por pantalla la información solicitada en el enunciado.
 2 puntos

Ejercicio 3: Mapa de colores.

  • Se crea y rellena cuna estructura de tipo Map usando como claves los colores del array y asignándoles el valor cero.
  • Se crea y rellena con nombres de colores aleatorios una estructura de tipo lista tal y como se indica en el enunciado.
  • Se actualiza el Map  con el número de veces que aparece cada color.
  • Se muestra por pantalla el resultado correcto y en un formato apropiado.

2 puntos

Ejercicio 4: Asignación de turnos (I).

  • Se declara correctamente la estructura mapTurnosPorFecha de tipo Map cuyas claves sean de tipo fecha y sus elementos de tipo lista de cadenas.
  • Se crea y rellena una estructura de tipo Map usando como claves las fechas indicadas en el enunciado y asignándole a cada clave una lista vacía.
  • Se van recorriendo las claves del Map y asignando las tres personas correspondientes en la lista asociada a cada fecha teniendo en cuenta que la primera persona asignada en una fecha no será asignada para el día siguiente y por tanto habrá que avanzar una posición más en el array de nombres.
  • Se va mostrando correctamente por pantalla el proceso de asignación.
  • Se muestra por pantalla el contenido final de la estructura mapTurnosPorFecha en el formato indicado en el enunciado. Esto es, colocando en una línea diferente la lista de personas asociada a cada día.

2 puntos

Ejercicio 5: Asignación de turnos (II).

  • Se declara correctamente la estructura mapTurnosPorPersona de tipo Map cuyas claves son de tipo cadena y sus elementos de tipo lista de fechas.
  • Se exploran todos los elementos de la estructura Map de turnos por fechas (creada en el ejercicio anterior).
  • Para cada elemento de la estructura Map de turnos por fechas, que será de tipo lista de nombres de persona, se recorre cada elemento de esa lista.
  • Si se observa que un nombre de persona aún no es una clave de la nueva estructura mapTurnosPorPersona, se crea una nueva entrada donde la clave será el nombre de esa persona y su valor asociado una lista de fechas.
  • Por cada elemento encontrado en la lista de personas asociada a cada fecha, se añadirá en la nueva estructura mapTurnosPorPersona una nueva entrada en la lista de fechas asociada a esa persona.
  • Se indican por pantalla las acciones que se van realizando cada vez que se encuentra a una persona por primera vez y/o cuando se asigna a una persona una determinada fecha.
  • Se muestra por pantalla el contenido final de la estructura mapTurnosPorPersona en el formato indicado en el enunciado. Esto es, colocando en una línea diferente la lista de fechas asociada a cada persona.

1 punto

Ejercicio 6: Ordenación de productos.

  • Se declara, instancia y rellena una lista de objetos de tipo Producto tal y como se indica en el enunciado.
  • Se han implementado correcta y apropiadamente las siguientes clases tal y como se pide en el enunciado:
    • ComparadorProductosPorNombre,
    • ComparadorProductosPorCaducidad,
    • ComparadorProductosPorPrecio.
  • Se ordena correcta y apropiadamente la lista por nombres de producto, por caducidades de producto y por precios de producto.
  • Se muestra por pantalla el contenido de la lista tras cada ordenación tal y como se indica en el enunciado.
 2 puntos
